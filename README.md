一、硬件层：I2C 与 GPIO 的基础适配

I2C 引脚与速率匹配

I2C1 配置为100000Hz（标准模式），引脚映射到 PB6（SCL）、PB7（SDA），并配置为GPIO_MODE_AF_OD（开漏复用）—— 这是 I2C 通信的标准配置，符合 VL53L0X 传感器的 I2C 电气特性（传感器支持 100kHz/400kHz I2C）。

传感器地址0x52是 7 位地址0x29左移 1 位（带读写位），符合 VL53L0X 的默认 I2C 地址规范，地址匹配是通信的前提。

XSHUT 引脚（PA5）的处理

PA5 配置为推挽输出，虽然代码里初始化阶段没显式操作 XSHUT，但传感器上电后默认处于工作状态（XSHUT 悬空 / 高电平），因此无需额外拉低 / 拉高也能通信；同时 PA5 复用为 LED，不影响传感器核心通信（只要硬件上没有物理冲突）。

二、I2C 通信层：鲁棒性设计保证通信不中断

代码中VL_Read/VL_Write函数做了I2C 状态检测 + 复位 逻辑：

if(HAL_I2C_GetState(&hi2c1) != HAL_I2C_STATE_READY)
{
  HAL_I2C_DeInit(&hi2c1);
  HAL_Delay(10);
  MX_I2C1_Init();
}

若 I2C 总线卡死（比如传感器未响应导致总线忙），会重新初始化 I2C1，避免因单次通信失败导致程序卡死，这是 “能跑通” 的关键鲁棒性设计。

读写超时时间（200ms/500ms）足够覆盖传感器的响应延迟，不会因超时导致误判通信失败。

三、传感器初始化：寄存器配置匹配 VL53L0X 工作逻辑

VL5310X_Init函数的寄存器配置是传感器能输出距离的核心：
先复位传感器：通过0x88/0x80/0xFF等寄存器组合完成软复位，确保传感器回到初始状态。

配置测距参数：
配置0x03=0x01（启动测距模式）、0x32=0x01（使能测量）、0x29/0x2A/0x31等寄存器设置时序 / 增益参数 —— 这些寄存器值是 VL53L0X “单次测距模式” 的经典配置（虽代码里写的是 VL5310X，但寄存器映射和 VL53L0X 兼容）。

ID 校验（0xC0 寄存器）：读取传感器 ID 寄存器，虽未做校验逻辑，但能通过串口打印确认传感器是否在线，为调试提供依据。

四、数据读取逻辑：符合传感器输出规范

VL5310X_GetDistance的流程完全匹配 VL53L0X 的测距流程：
启动测距：写0x00=0x01触发单次测距；

等待数据就绪：读0x13状态寄存器（虽未做 “就绪位判断”，但通过HAL_Delay(50)延时足够传感器完成一次测距）；

读取距离值：距离值存储在0x1E（高8位）和0x1F（低8位），拼接为 16 位数值（单位 mm）—— 这是 VL53L0X 距离输出的标准寄存器地址。
